# 中普咨询 · 报告排版标准规范

**文件编号：ZP-STD-LAYOUT-001**
**编制部门：创意部（方文君）/ 运营部（徐明辉）**
**生效日期：2024年**
**审核链：方文君（创意）→ 徐明辉（运营）→ CEO**

---

## 一、总则

本规范适用于中普咨询所有正式交付物（调研报告、行业分析、战略咨询等）。所有报告须严格遵循本规范排版，运营部（四级审核）负责检查执行情况。

## 二、页面布局

| 项目 | 标准值 | 备注 |
|------|--------|------|
| 纸张 | A4（210mm × 297mm） | 标准 |
| 页边距 | 上2.5cm 下2.5cm 左3.0cm 右2.5cm | 左侧预留装订 |
| 正文行距 | 1.5倍 | 可读性优先 |
| 段落间距 | 6pt | 段间自然分隔 |
| 首行缩进 | 2字符 | 仅正文段落 |

## 三、字体体系

### 3.1 字体选择

| 用途 | 中文字体 | 英文字体 | 备注 |
|------|---------|---------|------|
| 报告标题（封面） | 楷体（KaiTi）**18pt 加粗** | LibertinusSerif-Bold 18pt | 居中 |
| 章标题（##） | 楷体（KaiTi）**15pt 加粗** | LibertinusSerif-Bold 15pt | 每章新起一页 |
| 节标题（###） | 楷体（KaiTi）**13pt 加粗** | LibertinusSerif-Bold 13pt | 段前12pt |
| 小节标题（####） | 楷体（KaiTi）**11pt 加粗** | LibertinusSerif-Bold 11pt | 段前6pt |
| **正文** | 楷体（KaiTi）**11pt** | LibertinusSerif-Regular 11pt | 主要字体 |
| 表格内容 | 楷体（KaiTi）**9pt** | LibertinusSerif-Regular 9pt | 表头可加粗 |
| 代码/化学式 | DejaVuSansMono 8.8pt | DejaVuSansMono 8.8pt | 等宽 |
| 脚注/注释 | 楷体（KaiTi）**8pt** | LibertinusSerif-Regular 8pt | — |
| 页码 | Times New Roman 9pt | — | 页脚居中 |

## 四、目录规范

| 项目 | 标准 |
|------|------|
| 深度限制 | 仅到##级（`--toc-depth=2`） |
| 页数控制 | ≤2页 |
| 格式 | 自动生成（禁用手动锚点链接） |
| 位置 | 封面页之后、执行摘要之前 |

## 五、封面页模板

封面应包含（自上而下）：
1. 公司全称：**中普咨询 · ZhongPu Consulting**
2. 保密标记：🔴 绝密 / 🟡 机密 / 🟢 内部使用
3. 报告标题（中文 + 英文副标题）
4. 报告编号：`ZP-{年份}-R{序号}`
5. 编制部门及主笔人
6. 审核链：部门→数据→情报→运营→CEO
7. 报告日期 + 版本号

## 六、表格规范

### 6.1 格式要求
- 表头：深蓝背景（#1F4E79）+ 白色加粗字
- 行交替色：#F2F6FC / #FFFFFF
- 字号：9pt
- 边框：0.5pt #CCCCCC
- 编号：`Table X-Y`（X=章号，Y=章内序号）
- 标题：表格上方居中

### 6.2 注意事项
- 表格行之间**禁止空行**（否则 typst 拆分为多个单表）
- 避免 >12 行长表，超出需拆分
- 年份标注使用 `2025(估)` 而非 `2025E`（避免被当指数）
- 数字范围使用 `-`（连字符）而非 `~`（避免被当下标）

## 七、图表规范

| 项目 | 标准 |
|------|------|
| 图表编号 | `Figure X-Y`（X=章号，Y=章内序号） |
| 标题位置 | 图表下方居中 |
| 分辨率 | ≥150 DPI |
| 格式 | PNG / SVG 优先 |
| 配色 | 统一深蓝+金色品牌色系 |

## 八、页眉页脚

| 项目 | 内容 |
|------|------|
| 页眉 | 左：中普咨询 · 报告名称  右：密级标记 |
| 页脚 | 居中：页码（`1 / 28`） |
| 封面页 | 不显示页眉页脚 |

## 九、参考文档排版

- 参考文献按类别拆分（学术/行业/政策/企业），每类3-7条
- 每条含：编号、来源标题、发布单位、日期、URL（如有）
- 参考文献前**不加分页符**，自然承接结论章

## 十、生成规范（技术实现）

### 10.1 推荐工具链（按优先级）
| 优先级 | 工具 | 适用场景 |
|:------:|------|---------|
| 1 | pandoc + typst | 标准报告（首选，中文原生支持） |
| 2 | pandoc + xelatex | 复杂表格/多图报告 |

### 10.2 Markdown 源文件要求
- 使用 `##` 为章标题，`###` 为节标题
- 表格用标准 Markdown `|` 语法
- 不要使用手动锚点链接 `[text](#id)`
- 不要使用 `~` 表示范围（用 `-`）
- 不要使用 `2025E` 表示预估（用 `2025(估)`）
- 不要使用 emoji 符号（某些字体不支持）
- 每章末尾加 `\pagebreak` 分页（参考文献前不加）

### 10.3 编译命令（模板）
```bash
pandoc report.md -o report.pdf \
  --pdf-engine=typst \
  --toc \
  --toc-depth=2 \
  --metadata title="中普咨询 · 报告标题"
```

### 10.4 文件命名
```
{序号}-{报告名}-V{版本}.md
{序号}-{报告名}-V{版本}.pdf
```

## 十一、五级审核中的排版检查清单（运营部用）

运营部（四级审核）须逐项检查以下内容：

- [ ] 封面信息完整（公司名、标题、编号、密级、日期、审核链）
- [ ] 目录≤2页，仅到##级
- [ ] 正文统一使用楷体11pt
- [ ] 英文统一使用LibertinusSerif
- [ ] 表格样式一致（深蓝表头+交替行）
- [ ] 表格行之间无空行
- [ ] 无 `2025E` 类标注（应为 `2025(估)`）
- [ ] 无 `~` 范围符号（应为 `-` 或 `≈`）
- [ ] 参考文献≥20条，分类型排列
- [ ] 页码连续，封面无页码
- [ ] 每个##章从新页开始（参考文献除外）
- [ ] 无孤页（页面内容<100字符的视为孤页）
- [ ] 文件命名合规
